# coding-skills
Skills for spec-driven development through GitHub issues and pull requests

## Documentation

These commands are documented in the following articles:

- [Three Commands to a Crude Orchestrator](https://meshintelligence.substack.com/p/three-commands-to-a-crude-orchestrator) — `make-work`, `gh-issue-push`, `do-work`: the planning/execution split
- [Two Claude Skills and a Worktree Rule](https://meshintelligence.substack.com/p/two-claude-skills-and-a-worktree) — `gh-issue-pop` and the worktree-per-issue rule
- [How to Code with GLM 5.2 on OpenCode](https://meshintelligence.substack.com/p/how-to-glm-52-on-opencode) — running the full loop on OpenCode with a cheap model

Experiment commands (`/exp-start`, `/exp-stop`) run local-only by default, or
in a subtree-backed remote sync mode (`--remote <owner/repo>`) that keeps a
two-way `git subtree` boundary with a second repository.

Assistant configuration is canonical under `.claude/` and mirrored to
`.cursor/` (commands and skills), `.opencode/` (commands and skills),
`.codex/` (repository instructions, prompts, and skills), and `.github/`.
The `.codex/` mirror provides a self-contained `AGENTS.md`, complete command
workflows under `prompts/`, and a Codex-discoverable `SKILL.md` wrapper for
every command alongside the reusable skills. All internal paths stay under
`.codex/skills/`; the generated root `AGENTS.md` makes the same instructions
discoverable by Codex in this repository. The `.github/` mirror is self-contained: each prompt inlines
the full command workflow, `copilot-instructions.md` inlines the agent
instructions and rules, and the skills are copied under `.github/skills/`
with every path reference rewritten to stay inside the subtree. Nothing
under `.github/` points back at `.claude/`, so the whole tree works as a
bare symlink — `ln -s .../.github .github` drops the configuration into
another repository for GitHub Copilot with no dangling references.
Mirrors are generated by `scripts/sync-mirrors.sh` (`--check` reports
drift). See [.claude/README.md](.claude/README.md).

The skills' Python dependencies are managed by [pixi](https://pixi.sh/): a
`pixi.toml` and `pixi.lock` ship inside each agent directory, and
`scripts/ensure-env.sh` provisions the locked environment on repo open
(installing pixi itself if absent). Because the manifest travels with the
symlinked agent directory, a fresh machine needs no manual setup — the
preflight materializes the exact environment. API keys (`SERPAPI_KEY`,
`ANTHROPIC_API_KEY`, optional `SEMANTIC_SCHOLAR_API_KEY`) are supplied via the
environment, not pixi.
